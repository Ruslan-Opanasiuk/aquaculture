# Конвенції коду

Конкретні правила під цей стек. Якщо щось суперечить дисципліні з `CLAUDE.md` —
перемагає дисципліна.

## Компоненти
- Тільки функціональні компоненти + хуки. Класів немає.
- Один компонент = один файл `.jsx`. Назва файлу = назва компонента (`ProductCard.jsx`).
- Новий компонент — лише якщо не можна розширити існуючий. Спершу глянь у `components/`.
- Пропси прості й явні. Не передавай «мегаоб'єкти», якщо потрібні 2 поля.

## Кнопки
- **Стрілка-кнопка** (`components/ActionArrowButton.jsx`) — дефолтний вибір для дій,
  які стоять у контентному блоці поруч з іншою інформацією (ціна, підсумок, картка
  товару, breadcrumb-контекст). Не важливо, головна це дія екрана чи ні — переглянути
  товар, додати в кошик, оформити замовлення, повернутись до кошика — все це стрілка.
- **Суцільна pill-кнопка** (без стрілки) — лише для:
  1. Сабміту форми — фінальна дія форми з кількома полями (checkout, wholesale-анкета,
     newsletter).
  2. Самостійного CTA на hero-банері — кнопка єдиний інтерактивний елемент на великому
     фото-тлі, без сусіднього контенту (Home hero, WholesaleBanner, NotFound).
- Різниця не у «важливості» дії, а в контексті: контент-блок → стрілка; форма або
  порожній hero-banner → pill.
- `ActionArrowButton`: `size="md"` (44px, за замовчуванням) для самостійних кнопок,
  `size="sm"` (36px) — для контекстів з обмеженим місцем (кнопка всередині інпута).
  `tone="footer"` — для темного фону футера (зливається з фоном, hover — лише opacity,
  колір іконки не міняється).

## Стилі
- **Тільки Tailwind CSS 4** або CSS custom properties. Inline-стилів немає.
- Глобальні токени/змінні — у `src/index.css`. Кольори/відступи — через токени, не магічні числа.
- Дизайн незмінний: мінімалізм, багато «повітря», великі відступи, чітка ієрархія,
  преміальність без пафосу. Фокус на продукті, не на кількості.

## Дані
- Джерело правди — `src/data/` (`catalogData`, `caviarPackages`, `homeFeatures`, `seoConfig`).
- **Не дублюй і не хардкодь** дані в компонентах, якщо вони є в `data/`.
- Логіка знижок — незмінна, живе в `components/discount.jsx`:
  від 3кг → 7%, від 6кг → 14%, від 12кг → 21%, від 24кг → 28%.

## Стан
- Кошик — Zustand (`store/cartStore.jsx`). Не дублюй стан кошика в локальному `useState`.
- Локальний UI-стан (відкрито/закрито, інпут) — `useState`, не Zustand.

## Зображення
- Завжди WebP зі `srcSet` 1x/2x (retina). Генерація — `node src/optimize.js`,
  один скрипт на 4 категорії `raw/<категорія>/` → `optimized/<категорія>/`:
  - `products/` — фото товарів (банка/кришка). Квадрат 512/1024px, contain,
    lossy WebP (q90, alpha збережена) — реальне фото, lossless тут роздуває
    вагу в рази без приросту якості.
  - `grid/` — графічні іконки переваг/каруселі на Home. Квадрат 256/512px,
    contain, lossless — лінійна графіка стискається компактно без втрат.
  - `banners/` — повнокадрові фото-банери (hero, атмосферні). Пропорційний
    resize по ширині (оригінал = "2x", половина = "1x"), lossy q84, без
    підвищення чіткості (артефакти на фото).
  - `partners/` — лого партнерів у футер-каруселі. Resize по висоті 48/96px,
    lossless (плоска графіка з прозорістю).
  - Суфікс файлу завжди = реальний піксельний розмір (`-512.webp`,
    `-256.webp`), не умовний `-1x`/`-2x`.
- Нові оригінали кидай у відповідну `raw/<категорія>/`, ганяй `node src/optimize.js`,
  підключай з `optimized/<категорія>/`.

## Маршрути та SEO
- Маршрути — React Router 7: `/`, `/catalog`, `/product/:id`, `/cart`, `*`.
- SEO — `react-helmet-async` через `components/SEO.jsx`; конфіг у `data/seoConfig.jsx`.
- Кожна сторінка має title/description + canonical. Заглушки доменів — закрити перед запуском.

## Бекенд / email
- `src/server.jsx` — Express + Resend. Ключі тільки через env (`.env`, не в коді).
- Жодних секретів у репозиторії.
