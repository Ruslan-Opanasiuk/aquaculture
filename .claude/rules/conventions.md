# Конвенції коду

Конкретні правила під цей стек. Якщо щось суперечить дисципліні з `CLAUDE.md` —
перемагає дисципліна.

## Компоненти
- Тільки функціональні компоненти + хуки. Класів немає.
- Один компонент = один файл `.jsx`. Назва файлу = назва компонента (`ProductCard.jsx`).
- Новий компонент — лише якщо не можна розширити існуючий. Спершу глянь у `components/`.
- Пропси прості й явні. Не передавай «мегаоб'єкти», якщо потрібні 2 поля.

## Стилі
- **Тільки Tailwind CSS 4** або CSS custom properties. Inline-стилів немає.
- Глобальні токени/змінні — у `src/index.css`. Кольори/відступи — через токени, не магічні числа.
- Дизайн незмінний: мінімалізм, багато «повітря», великі відступи, чітка ієрархія,
  преміальність без пафосу. Фокус на продукті, не на кількості.

## Дані
- Джерело правди — `src/data/` (`catalogData`, `caviarPackages`, `homeFeatures`, `seoConfig`).
- **Не дублюй і не хардкодь** дані в компонентах, якщо вони є в `data/`.
- Логіка знижок — незмінна, живе в `components/discount.jsx`:
  від 3кг → 7%, від 6кг → 14%, від 12кг → 21%, від 24кг → 28%.

## Стан
- Кошик — Zustand (`store/cartStore.jsx`). Не дублюй стан кошика в локальному `useState`.
- Локальний UI-стан (відкрито/закрито, інпут) — `useState`, не Zustand.

## Зображення
- Завжди WebP зі `srcSet` 1x/2x. Генерація — `node src/optimize.js`
  (`src/assets/images/raw` → `src/assets/images/optimized`).
- Нові зображення кидай у `raw/`, ганяй optimize, підключай оптимізовані.

## Маршрути та SEO
- Маршрути — React Router 7: `/`, `/catalog`, `/product/:id`, `/cart`, `*`.
- SEO — `react-helmet-async` через `components/SEO.jsx`; конфіг у `data/seoConfig.jsx`.
- Кожна сторінка має title/description + canonical. Заглушки доменів — закрити перед запуском.

## Бекенд / email
- `src/server.jsx` — Express + Resend. Ключі тільки через env (`.env`, не в коді).
- Жодних секретів у репозиторії.
