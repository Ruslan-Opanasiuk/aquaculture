# Дисципліна рефакторингу

Ядро принципів — у `CLAUDE.md`. Тут — операційна процедура: коли, як і з якими
запобіжниками. Мета: рухатися малими безпечними кроками, не ламаючи робоче.

## Коли рефакторити
- **Так:** перед тим як додати фічу в заплутане місце (спершу прибери, потім додай);
  коли бачиш третє повторення (правило трьох); коли назва бреше про вміст.
- **Ні:** «бо некрасиво», без задачі; під час фіксу бага (спершу фікс, рефактор — окремо);
  коли немає способу перевірити результат.

## Безпечна процедура (кожен крок)
1. **Визнач межу.** Що саме змінюєш і що НЕ чіпаєш.
2. **Знайди залежності.** `grep` по імені компонента/функції/експорту — хто це імпортує.
3. **Один крок.** Рефактор без зміни поведінки АБО зміна поведінки. Не обидва разом.
4. **Перевір.** `npm run build` + прев'ю/очі. Поведінка не змінилась — добре.
5. **Зафіксуй.** Окремий коміт «refactor: …». Зміни поведінки — окремими комітами.

## Розділяй коміти
- `refactor:` — структура без зміни поведінки.
- `feat:` / `fix:` — зміна поведінки.
- Один логічний крок = один коміт. Не змішуй переміщення файлів зі зміною логіки.

## Червоні прапорці (код просить рефактор)
- Скопійований блок утретє → винеси у функцію/компонент/`data/`.
- Компонент > ~150 рядків або робить 3 різні речі → розбий.
- Пропс-дрилінг через 3 рівні → підніми у стор або контекст.
- Магічні числа/рядки повторюються → константа в `data/` або токен.
- Коментар пояснює *що* робить рядок (а не *чому*) → перепиши код зрозуміліше.

## Чого не робити
- Не «покращуй» код, який не стосується задачі (окрема задача — окремий PR).
- Не вводь абстракцію «на майбутнє» (YAGNI).
- Не залишай мертвий код «про всяк випадок» — видаляй, історія є в git.
- Не приймай великий діф, який не можеш прочитати очима повністю.

## Чек-лист перед тим, як сказати «готово»
- [ ] Збірка проходить (`npm run build`).
- [ ] Поведінка та сама (або зміна свідома й описана).
- [ ] Нічого зайвого не зачеплено; мертвий код прибрано.
- [ ] Дані не задубльовані (джерело — `data/`).
- [ ] Коротко: що змінив + що перевірити.
