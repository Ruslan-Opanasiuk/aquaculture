# CLAUDE.md — Aquaculture

Преміальний інтернет-магазин ікри. Кураторський підхід, не масмаркет: обмежений
асортимент, чистий дизайн, контрольована воронка. Три сегменти — роздріб (B2C),
корпоративні, гуртовики (верифікація через анкету).

> Особистий/бізнес-контекст — у `~/Desktop/CLAUDE.md`.
> Технічна пам'ять і рішення — у [`.claude/docs/architecture.md`](.claude/docs/architecture.md).

## Дисципліна розробки (Karpathy)
Це не стиль — це правила. Дотримуйся за замовчуванням.

- **Найменша зміна, що працює.** Спочатку змусь працювати → потім зроби правильно →
  лише потім швидко. Не навпаки.
- **Найкращий код — це відсутній код.** Видаляй сміливо: менше коду = менше багів.
  Кожен рядок мусить виправдати своє існування.
- **YAGNI + правило трьох.** Не будуй під уявне майбутнє. Абстракцію вводь на
  третьому повторенні, не раніше.
- **Одна зміна за раз.** Перевір (`npm run build` / очима / прев'ю) — і лише тоді
  наступна. Не накопичуй незавершене.
- **Рефакторинг ≠ зміна поведінки.** Не змішуй в одному кроці й одному коміті.
- **Читабельність > спритність.** Код читають у 10× частіше, ніж пишуть. Зрозуміле
  ім'я важливіше за хитрий трюк.
- **Короткий повідок для LLM.** Малі діфи, які можна прочитати очима повністю.
  Не приймай зміну, якої не розумієш.
- **Не ламай сусіднє.** Перед зміною — `grep` хто залежить від цього місця.

## Як працювати зі мною по цьому проєкту
